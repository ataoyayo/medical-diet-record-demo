# 医嘱限制饮食记录 Demo

## 项目简介

这是一个面向医嘱限制饮食场景的前端原型，用于记录每日饮食、估算蛋白质、碳水和脂肪摄入，并查看钠、钾、磷及其他医嘱提醒项。

## 当前版本定位

v0.3.0 public beta / experimental demo

用途：
- 功能体验测试
- 饮食记录流程测试
- 交互原型展示

不适用：
- 不能替代医生建议
- 不能用于临床决策
- 不能直接作为 CKD 个体化饮食处方

## 功能列表

- 三大营养素进度：蛋白质、碳水、脂肪
- 钠、钾、磷基础提醒
- 钙、液体、糖、嘌呤扩展提醒
- 单食材记录
- 整道菜记录
- 分食估算
- 用油估算
- 加入前预计
- 今日提醒
- 饮食记录归档抽屉
- 近 7 / 14 / 30 天饮食回看

## 使用方式

1. 打开 `index.html`。
2. 切换到“医嘱限制饮食模式”。
3. 在“我的饮食限制”中录入或编辑限制。
4. 选择食材、输入重量、设置分食和用油。
5. 可选择菜品方案，也可直接加入单食材。
6. 在“已摄入饮食”和“饮食记录”中查看当天与历史记录。

## 医疗免责声明

本项目不是医疗器械。

本项目不提供诊断、治疗或个体化营养处方。所有营养数据仅用于估算和功能测试。CKD 患者应根据医生或注册营养师建议管理饮食。

## 数据说明

食材营养数据为常见食材估算值。肉类默认按较常见家庭食用部位估算，牛肉默认按偏瘦牛肉估算；不同部位、品牌、熟重/生重、调味和烹饪方式会造成差异。用油量按烹饪方式默认估算，用户可以手动修改。钠、钾、磷、钙、限水、控糖、嘌呤等项目仅作为医嘱提醒，不生成自动饮食处方。

## 当前已知限制

- 不接入真实医疗系统。
- 不提供个体化营养处方。
- 钠、钾、磷等提醒以文本和风险提示为主，不做精确摄入量计算。
- 食材数据不是权威数据库，只适合原型测试。
- localStorage 存储在本机浏览器内，清理浏览器数据会丢失记录。
- 当前版本是静态前端页面，可直接部署到 GitHub Pages；数据仍保存在用户自己的浏览器 localStorage 中。

## 本地运行方式

无需安装依赖。直接用浏览器打开：

```bash
open index.html
```

如果浏览器限制本地文件，也可以启动一个简单静态服务器：

```bash
python3 -m http.server 8000
```

然后访问 `http://localhost:8000`。

项目使用 `./index.html`、`./style.css`、`./app.js` 等相对路径，不依赖后端服务，适合 GitHub Pages 静态部署。

## 反馈方式

公开测试阶段建议反馈：
- 哪一步让人困惑
- 哪个数字看起来不可信
- 哪个提醒太焦虑或不够明确
- 移动端是否出现遮挡、横向滚动或按钮难点
